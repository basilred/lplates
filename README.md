# LPlates (License Plates)

Современное и быстрое PWA-приложение на React для определения региона по автомобильному номеру или коду региона.

Приложение спроектировано как высокопроизводительный инструмент с премиальным дизайном, поддерживающий автоматический парсинг полных номеров и мультиязычный интерфейс.

## Основные возможности

- **Умный парсинг**: Введите просто код региона (например, `77`) или полный номер автомобиля (например, `А123БВ777` или `AA1234BB`). Приложение само извлечет нужную информацию.
- **Поддержка нескольких стран**:
  - 🇷🇺 Россия
  - 🇺🇦 Украина
  - 🇧🇾 Беларусь
  - 🇨🇿 Чехия
- **Интернационализация (i18n)**: Интерфейс доступен на 6 языках: Русский, Украинский, Белорусский, Английский, Чешский и Крымскотатарский.
- **PWA (Progressive Web App)**: Возможность установки на смартфон или компьютер, работа в оффлайн-режиме.
- **Высокая производительность**: 
  - Использование **React 19** и **Concurrent Rendering**.
  - Использование `useDeferredValue` для отзывчивости ввода при больших объемах данных.
- **Тёмная тема**: Полноценная поддержка светлой и тёмной тем оформления с автоматическим определением предпочтений системы ("System Theme"), а также ручным переключением между тремя состояниями (Light, Dark, System).
- **Премиальный дизайн**: Современный интерфейс с эффектами Glassmorphism, адаптивной версткой и поддержкой мобильных клавиатур. Изолированное управление состоянием компонентов для предотвращения лишних перерендеров.

## Технологический стек

- **Core**: `React 19`
- **Build Tool**: `Vite 6`
- **Language**: `TypeScript 5`
- **Performance**: `React Suspense`, `useDeferredValue`.
- **PWA**: `vite-plugin-pwa` для кэширования и установки.
- **Testing**:
  - `Vitest` — для модульных тестов логики парсинга.
  - `Playwright` — для сквозных (E2E) тестов интерфейса.
- **Deployment**: `gh-pages`

## Запуск локально

### Требования
- `Node.js` 18.0.0 или выше
- `npm`

### Установка
```bash
npm install
```

### Разработка
Запуск dev-сервера с hot reload:
```bash
npm run dev
```

### Тестирование
Запуск модульных тестов (Vitest):
```bash
npm test
```

Запуск E2E тестов (Playwright):
```bash
npm run test:e2e
# Или с UI интерфейсом
npm run test:e2e:ui
```

### Сборка и деплой
Сборка production-версии:
```bash
npm run build
```

Публикация на GitHub Pages:
```bash
npm run deploy
```

## Структура проекта

```text
.
├── e2e/                # Сквозные тесты (Playwright)
├── public/             # Статические ресурсы и манифест PWA
├── src/
│   ├── App/            # Основной компонент приложения (Layout и статика)
│   ├── components/     # UI компоненты (Input, List, LookupPanel, LanguageSwitcher, ThemeToggle)
│   ├── contexts/       # Контексты (LanguageContext, ThemeContext)
│   ├── hooks/          # Кастомные React-хуки (useRegionData, useTranslation)
│   ├── locales/        # Файлы переводов (json)
│   ├── utils/          # Утилиты, парсер номеров и countryUtils
│   ├── data.json       # База данных регионов и кодов
│   ├── index.css       # Глобальные стили и дизайн-система
│   ├── index.tsx       # Точка входа
│   └── sw.ts           # Service Worker (PWA)
├── playwright.config.ts # Конфигурация Playwright
├── tsconfig.json       # Конфигурация TypeScript
└── vite.config.ts      # Конфигурация Vite и PWA плагина
```

## Текущее состояние и качество
- **Тесты**: Логика парсинга покрыта модульными тестами на 100%. Основные сценарии использования проверены через E2E.
- **Производительность**: Минимальный размер бандла, оптимизированные перерендеры через декомпозицию состояния, использование отложенного ввода (deferred values).
- **UX**: Автоматический перевод в верхний регистр, поддержка ввода кириллицей/латиницей там, где это уместно.
